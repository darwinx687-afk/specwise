# SpecWise v0.2.0-preview.0 Release Notes Draft

Status: Release notes for the v0.2.0-preview.0 GitHub pre-release.

These notes were used for the v0.2.0-preview.0 GitHub pre-release.

This preview is local-first, review-required, and source-install only. It does not publish npm.

## Highlights

- Adds a second realistic synthetic example pack for a legacy CRM follow-up admin workflow.
- Improves deterministic draft quality while keeping generated drafts review-required.
- Improves dependency-free CSV entity and field inference.
- Improves first-run CLI errors, help output, and success summaries.
- Simplifies first-time documentation with a Start Here map and command decision tree.

## What Changed Since v0.1.0-preview.0

### New Example Pack

- Added `examples/legacy-crm-follow-up` with synthetic mock screenshot text, CSV fixtures, messy bilingual notes, curated expected output, evidence map, and a `review_required` buildability report.
- Added validation coverage for the CRM expected-output spec-pack.

### Deterministic Draft Heuristic Polish

- Improved module, workflow/status, role/action/permission, open question, and evidence ID heuristics.
- Added `scripts/check-draft-quality.mjs` for staff and CRM deterministic draft smoke checks.

### CSV Field Inference Polish

- Improved dependency-free CSV entity names, field types, semantic hints, required hints, relationship hints, and observed enum/status values.
- Added field-level Markdown output for review.
- Added `scripts/check-field-inference.mjs`.
- XLSX remains inventory-only and design-only; no XLSX parser or parser dependency was added.

### CLI/DX Polish

- Improved first-run error guidance for unknown commands, missing arguments, missing paths/files, and output folder protection.
- Preserved review-required and no-provider/no-agent wording in command output.
- Added `scripts/check-cli-dx.mjs` and wired it into `npm test`.

### First-time Docs Simplification

- Added `docs/start-here-v0.2.md`.
- Added `docs/command-decision-tree-v0.2.md`.
- Updated README, first-run guide, examples guide, docs index, roadmap, and v0.2 planning docs for clearer first-time navigation.

## Safety Boundaries

- No real AI provider calls.
- No prompt runner.
- No OCR or vision.
- No Web UI.
- No coding agent execution.
- No generated application code.
- No auto-apply patch.
- No final spec-pack generation.
- No npm publish.
- `package.json` remains `private: true`.

## Known Limitations

- Provider-backed extraction is not available in the v0.1/v0.2 baseline.
- OCR, vision, PDF parsing, Figma import, database connection, and Web UI are not included.
- XLSX binary parsing is not included.
- Deterministic drafts are heuristic and require human review.
- Handoff packs are context only and do not authorize implementation.

## Upgrade Notes

- Source users can pull the `v0.2.0-preview.0` preview tag.
- No npm upgrade path is available because npm publication is intentionally skipped.
- Existing v0.1 local commands remain local-first and review-required.

## Validation Commands

```bash
find bin src scripts -name "*.mjs" -print0 | xargs -0 -n1 node --check
node scripts/check-cli-dx.mjs
node scripts/check-field-inference.mjs
node scripts/check-draft-quality.mjs
npm test
npm run smoke
node bin/specwise.mjs --help
node bin/specwise.mjs doctor
```

## Not Included Yet

- Provider-backed extraction.
- OCR or vision processing.
- Web UI.
- Coding agent execution.
- Automatic patch application.
- Final spec-pack generation.
- npm publication.
