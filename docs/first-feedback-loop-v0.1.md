# First Feedback Loop v0.1

Status: Phase 17 feedback operations note. This is not an implementation plan.

## Goal

Collect early feedback from developers, FDEs, PMs, AI coding users, and people working with messy legacy systems.

The goal is to learn where SpecWise v0.1.0-preview.0 is clear, confusing, useful, or incomplete before choosing v0.2 scope.

## Feedback Channels

- GitHub Issues
- LinkedIn comments
- X/Twitter replies
- 小红书评论
- 掘金评论
- Direct messages, if the project owner chooses

Primary public entry points:

- Repo: https://github.com/darwinx687-afk/specwise
- Release: https://github.com/darwinx687-afk/specwise/releases/tag/v0.1.0-preview.0
- Feedback guide: feedback-guide-v0.1.md

## Triage Categories

- CLI usability

## Weekly Triage Checklist

- [ ] Review new issues.
- [ ] Label issues.
- [ ] Identify repeated confusion.
- [ ] Update docs if needed.
- [ ] Collect v0.2 candidates.
- [ ] Do not start implementation without planning.

## Stop Rules

- Do not request or store sensitive business materials in public issues.
- Do not start v0.2 implementation during Phase 17.
- Do not change safety boundaries without a reviewed plan.
- Do not claim real AI extraction, agent calls, or automatic app generation exists.
