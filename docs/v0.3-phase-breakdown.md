# v0.3 Phase Breakdown

Status: Phase 27D phase breakdown with workflow inference completion note.

This document breaks the proposed v0.3 baseline into later phases. It does not start implementation.

Phase 27A CI implementation was owner-approved and completed as a minimal validation-only workflow.
Phase 27B review/apply-plan usability polish was completed with local-only checks.
Phase 27C handoff pack readability polish was completed with local-only checks.
Phase 27D deterministic workflow inference polish was completed with local-only checks.

## Phase 27A: GitHub Actions CI / Reliability Planning or Implementation

Status: Completed.

### Goal

Make repository validation easier to run consistently after v0.2.

Completion note:

Phase 27A added `.github/workflows/ci.yml` for pull requests and pushes to `main`. The workflow runs validation only, uses read-only contents permission, uses only official GitHub actions, and does not publish npm, create tags, create GitHub releases, use secrets, deploy, call AI providers, call coding agents, generate application code, generate final spec-packs, or auto-apply patches.

### Allowed Changes

- Document CI check design.
- Add a minimal GitHub Actions workflow for local validation commands after owner approval.
- Keep checks limited to syntax, tests, smoke, and hygiene-friendly validation.

### Disallowed Changes

- No workflow expansion without owner approval.
- No npm publish.
- No release automation.
- No secrets.
- No external services.
- No real AI provider calls.

### Candidate Files

- `docs/v0.3-implementation-plan.md`
- `docs/v0.3-test-strategy.md`
- `.github/workflows/ci.yml`

### Acceptance Criteria

- CI scope is documented before implementation.
- Any future workflow avoids npm publish and releases.
- No secrets are required.
- Existing local checks remain the source of truth.
- CI safety documentation is linked.

### Test Commands

- `find bin src scripts -name "*.mjs" -print0 | xargs -0 -n1 node --check`
- `npm test`
- `npm run smoke`

### Hygiene Checks

- No workflow contains tokens, secrets, publish steps, or release steps.
- No dependency folders or temporary artifacts are committed.

### Rollback Notes

If CI scope expands toward publish or release automation, revert workflow changes and keep the phase planning-only.
