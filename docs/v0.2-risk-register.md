# v0.2 Risk Register

Status: Phase 22F risk register with readiness mitigation notes.

| Risk | Impact | Mitigation | Owner / Review Point |
|---|---|---|---|
| Scope creep into AI provider runtime | Adds trust, secret, and network risks before deterministic baseline improves | Keep provider runtime out of v0.2 baseline; require separate approval | Phase 21/22 planning review |
| XLSX support requiring dependency | Breaks dependency-free posture or adds maintenance risk | Keep XLSX design-only unless dependency-free path is confirmed or dependency decision is documented; Phase 22C kept XLSX inventory-only and added no parser dependency | Phase 22C completed; review again before any XLSX parser work |
| Examples becoming too artificial | Reduces demo value and hides real workflow complexity | Use realistic synthetic materials with messy notes, CSV, and mock screenshot text | Phase 22A review |
| Heuristics overclaiming facts | Could turn uncertain material into false requirements | Keep assumptions and open questions visible; compare evidence coverage; Phase 22B adds draft quality checks for required open questions, review-required status, and readable evidence IDs | Phase 22B completed; continue review in later heuristic changes |
| CSV relationship hints overclaiming schema facts | Could make sampled `*_id` fields look like confirmed foreign keys | Keep relationship hints low-confidence, review-required, and phrased as possible references; Phase 22C adds field smoke checks for relationship targets without marking them final | Phase 22C completed; continue review in later field inference changes |
| CLI output implying finality | Users may think generated drafts are final specs | Preserve `review_required` language in success and error output; Phase 22D keeps review-required summaries and adds CLI DX checks for first-run error guidance | Phase 22D completed; continue review in later CLI changes |
| Docs becoming too large | First-time users may not find the safe path | Add concise navigation and keep detailed docs linked; Phase 22E added Start Here and command decision tree docs | Phase 22E completed; continue review |
| Handoff pack being mistaken as implementation instruction | Users may treat context skeleton as approval to code | Keep context-only and no-agent-call boundaries visible; Phase 22F release readiness docs preserve no-agent and no-code-generation boundaries | Phase 22F completed; continue review |
| Release readiness mistaken for a release | Users may think `v0.2.0-preview.0` exists before approval | Keep release notes as draft, keep CHANGELOG section Unreleased, and state that tag/release/npm publish are not created in Phase 22F | Phase 22F completed; Phase 23 requires owner approval |

## Review Cadence

- Review this register before each Phase 22 sub-phase.
- Add risks when a change touches safety boundaries.
- Do not close risks only because tests pass.
- Treat public feedback as a signal to update mitigations.
