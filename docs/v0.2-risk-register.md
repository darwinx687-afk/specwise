# v0.2 Risk Register

Status: Phase 21 planning document.

| Risk | Impact | Mitigation | Owner / Review Point |
|---|---|---|---|
| Scope creep into AI provider runtime | Adds trust, secret, and network risks before deterministic baseline improves | Keep provider runtime out of v0.2 baseline; require separate approval | Phase 21/22 planning review |
| XLSX support requiring dependency | Breaks dependency-free posture or adds maintenance risk | Keep XLSX design-only unless dependency-free path is confirmed or dependency decision is documented | Phase 22C review |
| Examples becoming too artificial | Reduces demo value and hides real workflow complexity | Use realistic synthetic materials with messy notes, CSV, and mock screenshot text | Phase 22A review |
| Heuristics overclaiming facts | Could turn uncertain material into false requirements | Keep assumptions and open questions visible; compare evidence coverage | Phase 22B review |
| CLI output implying finality | Users may think generated drafts are final specs | Preserve `review_required` language in success and error output | Phase 22D review |
| Docs becoming too large | First-time users may not find the safe path | Add concise navigation and keep detailed docs linked | Phase 22E review |
| Handoff pack being mistaken as implementation instruction | Users may treat context skeleton as approval to code | Keep context-only and no-agent-call boundaries visible | Phase 22F review |

## Review Cadence

- Review this register before each Phase 22 sub-phase.
- Add risks when a change touches safety boundaries.
- Do not close risks only because tests pass.
- Treat public feedback as a signal to update mitigations.
