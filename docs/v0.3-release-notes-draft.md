# SpecWise v0.3.0-preview.0 Release Notes Draft

Draft release notes for a future v0.3 preview.

This document is preparation only. The `v0.3.0-preview.0` tag and GitHub pre-release still require explicit owner approval in a later phase.

## Highlights

- Validation-only GitHub Actions CI.
- Clearer review and manual apply-plan outputs.
- More readable handoff packs with stronger safety boundary wording.
- Deterministic workflow/status inference improvements.
- Consolidated first-time and maintenance documentation paths.

## What Changed Since v0.2.0-preview.0

SpecWise v0.3 preview readiness focuses on reliability, review usability, handoff readability, deterministic workflow inference, and documentation navigation.

## CI / Reliability

- Added a validation-only GitHub Actions workflow for pushes and pull requests.
- Added CI safety documentation.
- Preserved local-first checks as the source of truth.
- CI does not publish packages, create tags, create GitHub releases, or use secrets.

## Review / Apply-plan Usability

- Review reports now group safe carry-forward items, business confirmation items, deferred items, rejected items, and blocked readiness reasons.
- Manual apply plans now include clearer priority, owner hints, high-risk indicators, grouped blocked items, and a Before Editing checklist.
- `npm run review:quality` covers the local review/apply-plan fixture flow.

## Handoff Readability

- Generated handoff packs now include clearer README navigation, non-negotiable boundaries, grouped open questions, implementation boundary notes, machine artifact explanations, and improved agent-specific boundary sections.
- `npm run handoff:quality` covers the local handoff fixture flow.

## Deterministic Workflow Inference

- Draft generation now surfaces observed status labels, transition candidates, branch/exception candidates, and open workflow questions more clearly.
- Ambiguous reopened, rejected, approved, and edited-after-submit behavior remains review-required.
- `npm run workflow:quality` covers staff and CRM draft workflow checks.

## Docs Consolidation

- README now has clearer documentation paths.
- Start Here, First-run Guide, Command Decision Tree, Docs Index, Docs Maintenance Guide, and Docs Status Map make it easier to choose the right document.
- Historical planning docs remain findable without becoming the first reading path.

## Safety Boundaries

- No default provider calls.
- No prompt runner.
- No OCR, vision, or Web UI.
- No coding-agent invocation.
- No application code generation.
- No auto-apply patch behavior.
- No final spec-pack generation.
- npm publication remains skipped.
- `package.json` remains private.

## Known Limitations

- Deterministic drafts remain heuristic and review-required.
- XLSX files are recognized in inventory metadata only.
- Provider-backed extraction remains outside the current preview boundary.
- Handoff packs remain context only.
- Final spec-pack generation is not included.

## Upgrade Notes

- Source checkout remains the supported path.
- Run `npm test` and `npm run smoke` after pulling.
- Existing commands keep their current behavior.
- No migration step is required for existing examples.

## Validation Commands

```bash
find bin src scripts -name "*.mjs" -print0 | xargs -0 -n1 node --check
node scripts/check-workflow-inference.mjs
node scripts/check-handoff-readability.mjs
node scripts/check-review-apply-usability.mjs
node scripts/check-cli-dx.mjs
node scripts/check-field-inference.mjs
node scripts/check-draft-quality.mjs
npm test
npm run smoke
node bin/specwise.mjs --help
node bin/specwise.mjs doctor
```

## Not Included Yet

- Provider-backed extraction runtime.
- OCR or vision processing.
- PDF parser.
- XLSX binary parsing.
- Database import.
- Web UI.
- Automatic patch application.
- Final spec-pack generation.
- npm package publication.
