## 2. Core Message

The first announcement should explain the gap SpecWise addresses:

- legacy rebuilds often begin with screenshots, spreadsheets, notes, permissions, and unclear workflows
- those materials need structure before implementation starts
- SpecWise v0.1 creates local, reviewable artifacts that expose facts, assumptions, open questions, evidence, acceptance criteria, and handoff boundaries
- this preview is intentionally scoped and asks for early workflow feedback

Short core message:

```text
SpecWise is a local-first preview for turning messy legacy business materials into reviewable, evidence-based spec packs before coding starts.
```

## 8. What Not To Claim

Do not claim that SpecWise v0.1:

- is ready for production use
- fully handles the end-to-end rebuild workflow
- clones a legacy system from screenshots
- automatically writes application code
- replaces product, engineering, or domain review
- has live AI extraction available
- calls coding tools or implementation agents
- guarantees spec quality

Keep the public message grounded in the current preview: local-first, dependency-free, deterministic baseline, evidence-first, review-required, and feedback welcome.
