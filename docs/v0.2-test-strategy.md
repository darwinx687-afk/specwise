## Command-Level Checks

Run these for every future implementation phase:

```bash
node bin/specwise.mjs --help
node bin/specwise.mjs doctor
node bin/specwise.mjs handoff validate --help
```

Run focused commands when touching examples, inventory, draft, or validation:

```bash
node bin/specwise.mjs validate examples/minimal/spec-pack
node bin/specwise.mjs validate examples/legacy-staff-evaluation/expected-output/spec-pack
node bin/specwise.mjs validate examples/legacy-crm-follow-up/expected-output/spec-pack
node bin/specwise.mjs validate examples/invalid/missing-buildability/spec-pack --expect-fail
node scripts/check-draft-quality.mjs
node scripts/check-field-inference.mjs
node scripts/check-cli-dx.mjs
```

Run the deterministic CRM draft check when touching examples or draft behavior:

```bash
node bin/specwise.mjs draft examples/legacy-crm-follow-up/input --out ./tmp/crm-draft --force
node bin/specwise.mjs validate ./tmp/crm-draft/spec-pack
rm -rf ./tmp/crm-draft
```

`npm test` includes expected-output validation, invalid fixture expected-fail validation, `scripts/check-draft-quality.mjs`, `scripts/check-field-inference.mjs`, and `scripts/check-cli-dx.mjs`.

## Manual Review Points

- New example data is synthetic.
- Open questions remain visible.
- Assumptions are not presented as facts.
- CLI output does not imply production readiness.
- Docs do not imply real AI extraction exists.
