# v0.2 Candidate Scoring v0.1

Status: Phase 19 planning aid. This does not implement v0.2.

This scoring is a planning aid, not an automated roadmap decision.

这个评分只是规划辅助，不是自动路线决策。

## Scoring Dimensions

| Dimension | Points | Question |
|---|---:|---|
| User pain clarity | 20 | Is the problem concrete, easy to understand, and tied to a real workflow? |
| Frequency of feedback | 20 | Has the signal appeared across multiple issues, comments, or channels? |
| Project positioning fit | 20 | Does it fit local-first, evidence-first, review-required SpecWise positioning? |
| Safety impact | 15 | Does it improve safety clarity or reduce risky misunderstanding? |
| Implementation size | 15 | Can it be scoped small enough for a preview iteration? |
| Demo value | 10 | Would it make the project easier to evaluate in public examples? |
| Total | 100 |  |

## Output Levels

| Score | Level | Meaning |
|---:|---|---|
| 80-100 | Strong v0.2 candidate | Strong signal for Phase 20 scope discussion |
| 60-79 | Consider for v0.2 | Worth discussing, but may need narrowing |
| 40-59 | Keep watching | Track more evidence before committing scope |
| 0-39 | Not now | Do not include in near-term scope |

## Review Rules

- Score repeated signals, not isolated preferences.
- Keep raw feedback links in `feedback-tracking-v0.1.md`.
- Use the weekly feedback report before choosing v0.2 scope.
- A high score is not permission to implement.
- A low score does not invalidate the feedback; it means the item is not ready for near-term scope.
- Re-score when new evidence appears.

## Not Allowed In Phase 19

Scoring must not trigger:

- v0.2 implementation
- real AI provider calls
- coding agent calls
- prompt runner behavior
- OCR, vision, or Web UI work
- final spec-pack generation
- auto-apply patch behavior
