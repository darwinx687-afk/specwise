v0.3 范围选择已记录在 [v0.3 Scope Selection](v0.3-scope-selection-v0.1.md)。Phase 25 只选择建议范围，不开始 v0.3 功能开发。

This document is a candidate pool, not a commitment.

## Current Baseline

SpecWise `v0.2.0-preview.0` is a local-first GitHub pre-release. It adds a second realistic mock example, deterministic draft polish, CSV field inference, CLI/DX improvements, and first-time docs simplification.

Current behavior remains review-required. Current commands do not call real AI providers, do not call coding agents, do not generate application code, do not auto-apply patches, and do not generate final spec-packs.

## Candidate v0.3 Directions

### 1. More Example Packs

Add more synthetic legacy-business scenarios to make the workflow easier to evaluate across domains.

Possible examples:

- inventory / stock movement admin
- internal approval workflow
- billing adjustment workflow

### 2. Better Deterministic Workflow Inference

Improve local heuristic extraction for statuses, transitions, roles, permissions, and open questions while preserving review-required output.

### 3. Review / Apply-plan Usability Polish

Make review reports, manual apply plans, blocked items, and revision checklists easier for humans to scan.

### 4. Handoff Pack Readability Polish

Improve context-only handoff pack structure, summaries, and agent-specific boundary notes without calling agents or creating implementation tasks.

### 5. Explicit AI Preview Runtime Gate Design

Design the opt-in gate for future provider-backed preview behavior without implementing provider runtime by default.

Any future runtime must keep explicit opt-in, secret safety, patch-only output, and human review boundaries.

### 6. Markdown/CSV-only File Parser Polish

Improve deterministic handling of Markdown and CSV inputs before expanding to higher-risk file types.

### 7. Optional npm Package Planning

Evaluate whether and when npm publication should happen.

This remains planning only. `package.json` currently remains `private: true`.

### 8. GitHub Actions CI Planning

Plan CI checks for validation, smoke tests, and hygiene without publishing packages or calling external model providers.

### 9. Documentation Consolidation After v0.2

Simplify the growing documentation map after v0.2 while keeping safety, limitation, release, and first-run docs easy to find.

### 10. Issue/Feedback Label Refinement

Refine future `v0.3 candidate` feedback tracking and issue labeling without automatically creating, relabeling, closing, or replying to issues.

## Deferred Unless Explicitly Approved

- real AI provider runtime
- OCR / vision
- PDF parsing
- Figma import
- database import
- Web UI
- coding agent calls
- auto-apply patches
- final spec-pack generation

## Inputs To Collect Before Scope Selection

- repeated GitHub issue themes
- repeated first-run confusion
- release feedback comments
- example requests
- safety-boundary confusion
- maintainer effort and dependency risk

## Scope Selection Gate

v0.3 scope should only be selected after feedback has been reviewed and candidate work is scored against safety, usefulness, maintenance cost, and dependency risk.

Phase 25 selects a proposed v0.3 baseline scope and does not start v0.3 implementation.

The proposed baseline is reliability, review/apply-plan usability, handoff readability, deterministic workflow inference, and public repo maintainability before real AI provider calls.
