# v0.3 Implementation Prerequisites

Status: Phase 27A prerequisite checklist.

This checklist must be reviewed before starting v0.3 implementation.

Phase 25 does not start v0.3 implementation.
Phase 26 adds implementation planning only and does not start v0.3 implementation.
Phase 27A adds CI validation only and does not start Phase 27B.

## Before Starting v0.3 Implementation

- [ ] Review current GitHub issues
- [ ] Confirm baseline scope
- [ ] Decide whether CI is planning-only or implementation
- [ ] Confirm no new dependencies unless separately approved
- [ ] Confirm npm publish remains out of scope
- [ ] Keep package private unless separately approved
- [ ] Preserve v0.2 validation scripts
- [ ] Preserve no-AI / no-agent boundaries
- [ ] Break work into small phases
- [x] Confirm CI implementation has separate owner approval before adding any `.github/workflows/` file
- [ ] Confirm review/apply polish remains manual-only
- [ ] Confirm handoff polish remains context-only
- [ ] Confirm workflow inference preserves uncertainty

## Boundary Checks

Before any implementation phase, confirm:

- no real AI provider runtime is being added by default
- no prompt runner is being added
- no OCR / vision is being added
- no Web UI is being added
- no coding agent calls are being added
- no generated application code is being produced
- no auto-apply patch behavior is being introduced
- no final spec-pack generation is being introduced
- no npm publication is being performed
- no GitHub Actions release automation is being introduced without separate approval
- no GitHub Actions workflow expansion is being introduced without separate approval

## Suggested Phase Split

- Phase 26: v0.3 implementation planning
- Phase 27A: GitHub Actions CI / reliability planning or implementation
- Phase 27B: Review/apply-plan usability polish
- Phase 27C: Handoff readability polish
- Phase 27D: Workflow inference polish
- Phase 27E: Docs consolidation
- Phase 27F: v0.3 release readiness

Each phase should have its own validation and stop rules.
