## 1. GitHub Repo Short Intro

SpecWise turns messy legacy business materials into reviewable, evidence-based spec packs for AI coding agents.

It is a local-first, dependency-free v0.1 preview for teams that need to turn screenshots, exports, and mixed notes into structured specs before asking an AI coding agent to build. The current workflow is review-required and evidence-first. It does not call real AI providers or coding agents yet.

## 2. LinkedIn English Post

I just published the public preview repo for SpecWise.

SpecWise turns messy legacy business materials into reviewable, evidence-based spec packs for AI coding agents.

The idea is simple: before handing an ambiguous legacy-system rebuild to an AI coding agent, make the uncertainty visible. Capture evidence, assumptions, permissions, workflows, acceptance criteria, open questions, and handoff boundaries first.

The v0.1 preview is intentionally modest:

- local-first
- dependency-free
- review-required
- evidence-first
- no real AI calls yet
- no coding agent calls

It includes a CLI, JSON schemas, a validator, deterministic draft generation, offline patch preview, human review reports, manual apply plans, and context-only handoff pack skeletons.

Repo: https://github.com/darwinx687-afk/specwise

## 5. 掘金中文技术帖开头

旧系统改造最难的一步，往往不是写代码，而是把“截图、表格、口头说明、权限规则、历史流程”整理成可以审查、可以追踪证据、可以交给开发者或 AI coding agent 阅读的规格。

SpecWise 把旧系统截图、表格和混乱业务说明整理成可审查、有证据链、可交给 AI coding agent 的开发规格包。

SpecWise v0.1 是一个本地优先、无依赖、review-required 的 CLI 预览版。它目前不调用真实 AI provider，不调用 coding agent，也不生成最终 spec-pack。它做的是更前置的一步：把材料盘点、草稿规格、证据映射、人工 review、manual apply plan 和 handoff skeleton 组织成一条可检查的工作流。

这个阶段的重点不是自动化一切，而是让不确定性先被看见。
