## Focused Checks

When touching review or apply-plan behavior:

```bash
node bin/specwise.mjs review validate examples/reviews/legacy-staff-evaluation.review-decisions.example.json
node bin/specwise.mjs apply-plan validate examples/apply-plans/legacy-staff-evaluation.manual-apply-plan.example.json
npm run review:quality
npm run smoke
```

When touching handoff output:

```bash
node bin/specwise.mjs handoff create examples/legacy-staff-evaluation/expected-output/spec-pack --apply-plan examples/apply-plans/legacy-staff-evaluation.manual-apply-plan.example.json --out ./tmp/handoff-check --target codex --force
node bin/specwise.mjs handoff validate ./tmp/handoff-check
rm -rf ./tmp/handoff-check
npm run handoff:quality
npm run smoke
```

When touching deterministic workflow inference:

```bash
node scripts/check-workflow-inference.mjs
node scripts/check-draft-quality.mjs
node bin/specwise.mjs draft examples/legacy-staff-evaluation/input --out ./tmp/draft-check --force
node bin/specwise.mjs validate ./tmp/draft-check/spec-pack
rm -rf ./tmp/draft-check
```

When touching documentation only:


## CI-Specific Boundary

CI is now approved for local validation commands only.
No network tests beyond GitHub's checkout and Node setup actions.
No provider tests.
No agent tests.
No npm publish tests.
No release automation tests in baseline.

Any future CI must run local validation commands only unless the owner approves additional behavior.

The current CI workflow does not install dependencies, cache `node_modules`, upload artifacts, use secrets, publish npm, create tags, or create GitHub releases.

## Regression Expectations

- Existing examples continue to validate.
- Invalid fixture continues to fail as expected.
- Smoke workflow continues to state that no provider, agent, application code, or final spec-pack is generated.
- Handoff validation continues to enforce safety phrases.
- Generated temporary outputs are cleaned before commit.
- No new dependency appears in `package.json`.

## Manual Review Points

- Review/apply-plan UX does not imply auto-apply.
- Handoff readability does not imply implementation authorization.
- Workflow inference keeps uncertain transitions as questions.
- Docs consolidation does not hide limitations.
- CI planning does not imply publish or release automation.
