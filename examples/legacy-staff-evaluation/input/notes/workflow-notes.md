# Workflow Notes

The expected evaluation workflow is approximately:

Draft -> Submitted -> Reviewed -> Approved

Some records may be rejected and returned for revision.

草稿记录由组长或评估专员创建。提交后进入审核流程。部门主管通常负责复核。部分记录可能被退回修改。

Known states:
- draft
- submitted
- reviewed
- approved
- rejected

Known actions:
- save draft
- submit
- review
- approve
- reject
- edit
- export

Unclear rules:
1. Can a submitted record be edited before review starts?
2. Can a rejected record return to draft?
3. Who can approve a reviewed record?
4. Is approval required before a record becomes final?
5. Are operation logs visible to all roles or only admins?
